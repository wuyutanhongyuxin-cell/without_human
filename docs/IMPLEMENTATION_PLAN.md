# 完整实施计划

## 阶段0：自动化执行环境

目标：建立可靠、隔离、可恢复的Claude与Codex协作通道。

- 清理不完整WSL仓库；
- 建立权限为700的专用项目目录；
- 从Windows已审计仓库注入干净副本；
- Claude仅允许读取和编辑该目录；
- 禁止宿主机、旧工作区、环境文件、Bash和GitHub凭据；
- 建立任务书、交付报告、补丁提取和返工协议；
- 保存每轮任务ID、基线提交、差异、测试和审查结果；
- 修正启动脚本BOM问题。

验收：Claude无法读取边界外文件；Codex能稳定取得修改；失败后可回到最近基线。

## 阶段1：可信证据库

- Document、EvidenceChunk、ImportJob数据模型；
- TXT、HTML、DOCX导入；
- PDF采用可靠解析器，失败必须显式报告；
- 来源、机构、文号、发布日期、有效状态和导入时间；
- SHA-256去重、分段和原文定位；
- 可引用、仅参考、禁止使用状态；
- 导入失败隔离、数据库迁移和管理界面。

验收：重复内容不重复入库；损坏文件不伪装成功；片段可追溯；禁止证据不进入引用结果。

## 阶段2：混合检索与引用验证

- 中文BM25、向量检索和RRF融合；
- 地区、机构、时间、文种和有效性过滤；
- 可插拔重排；
- 主张与证据绑定；
- 引用完整性、正确性和冲突检测；
- 证据不足拒答；
- 检索过程可解释。

验收：至少100题检索集；记录Recall@10、MRR和引用准确率；无依据结论被阻断。

## 阶段3：写作状态机

状态：资料录入 → 资料不足/可起草 → 已起草 → 待审 → 待修 → 可导出 → 已导出。

- 文种、任务要素、大纲、段落、主张和证据关联；
- 每段只能使用批准事实；
- 人工或规则锁定段落；
- 失败段落单独重写；
- 版本差异、回退和修改原因；
- 连续修复失败时停止。

验收：锁定段落不被修改；缺事实任务不能进入可起草；所有版本可回退。

## 阶段4：文种与硬审规则

- 覆盖15种法定公文；
- 支持方案、汇报、讲话和总结等常用材料；
- 行文关系、主送机关、文种选择；
- 请示一文一事、报告不得夹带请示等规则；
- 主体、任务、期限、机制、成果和验收检查；
- 政策、年份、数字、比例和机构名称核验；
- 规则版本化和单位自定义；
- 问题定位到字段、句子或段落。

验收：致命错误一票否决，不能被平均分抵消。

## 阶段5：正式DOCX排版

- 按GB/T 9704-2012建立可配置样式；
- A4版面、版心、标题、正文和层级标题；
- 发文字号、签发人、主送、附件、落款、日期、页码和版记；
- 表格、分页和字体缺失检查；
- Word/WPS兼容性测试；
- 渲染截图和版式回归；
- 导出前格式预检报告。

验收：Word/WPS无修复提示；无异常分页、字体丢失或溢出。

## 阶段6：安全与部署

- 本机配置页面；
- Key仅驻留进程；
- 云模型和本地模型统一接口；
- 展示外发数据范围；
- 备份、恢复、保留期和审计日志；
- 依赖锁定与软件物料清单；
- 默认只监听127.0.0.1；
- 可选完全离线部署。

验收：凭据不进入日志、数据库、前端、导出文件和Git。

## 阶段7：Benchmark与交付

- 200题公文规范确定性测试；
- 100题真实政务任务；
- 100题证据与幻觉对抗测试；
- 50题DOCX格式与渲染测试；
- 30题专家双盲实用性测试。

硬门槛：

```text
关键事实虚构率          0%
强制字段漏检率          0%
数字抄录准确率        100%
政策引用正确率        ≥98%
引用完整率            ≥95%
证据不足拒答召回率    ≥95%
```

完成后创建发布候选、执行干净环境复现、生成交付报告并打`v1.0.0`标签。
